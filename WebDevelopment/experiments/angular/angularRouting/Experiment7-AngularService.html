About the experiment

This experiment uses angularJS factory service to store the data common to the entire application. During page navigation the activity carried out in one page might not be informed to the other pages. In that case all pages will be inconsistent with the state of the application. So in order to have a consistent application, factory service is needed. Functionalites that are going to used across the entire application, data that is shared by multiple pages reside in the factory service.

The factory service is created giving it a meaningful name. Once the methods are created inside the service, they are exposed to the outside world using return. The module that wants to invoke the functionality inside the service should name the name exposed. It will then be internally mapped by the service to call the right method. Method creation inside the service is done as how we normally do in the controllers.

The experiment lets users to view the details of a particular product. Also they can view the reviews of a product adding their own. The reviews that they add will be stored in the service and fetched when visiting 'My Reviews' page. Similary the users favorites are maintained.


Code Snippet:

return {
fetchProducts: fetchProducts,
addToFavorites: addToFavorites,
getFavorites: getFavorites,
addToReviews: addToReviews,
getReviews: getReviews,
}
