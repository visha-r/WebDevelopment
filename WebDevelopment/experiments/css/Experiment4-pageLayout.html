CSS Experiment 4 - Page Layout using semantics & CSS styling

Webpage design

About the experiment

The number of visitors to a web page depends mostly on its layout design. Hence paying careful attention on choosing the web page layout design is very important. This experiment is about one one of a good layout designs. To start with a webpage design, the styling for header, navigation and footer elements have to be decided.

The following is the styling for header section. The header position must be made 'fixed' so it will not move even if the main content moves. Height of the header has been given as only 20% of the entire page.

Code Snippet:
header {
height: 20%;
width: 100%;
text-align: center;
background-color: lightblue;
position: fixed;
padding: 0px 30px 0px 30px;
}

The following is the styling for navigation section. The navigation section can be used to provide quick links to contents other than the ones in the main menu. The navigation section must also be made 'fixed' so it will not move if the main content moves.

Code Snippet:
nav {
width:15%;
height:70%;
position:fixed;
background-color:lightgreen;
margin-top:11%;
padding:100px 0px 100px 30px;
float:right;
}

The following is the styling for article section. The article element represents a component of a page that consists of a self-contained composition in a document. The position of article should not be made fixed as it should be able to flow freely. It must also have padding on top and left so it will not overlap with header and navigation links.

Code Snippet:
article {
width:80%;
height: 100%;
background-color: lightcyan;
margin:auto;
padding: 140px 0px 0px 250px;
}

Finally, the margin of body should be reset to 0 so the main contents when scrolled upwards will not be visible in the header section.
